## 一、试点范围

### 1.1 系统组成

| 组件 | 说明 |
|------|------|
| 服务端（factory-saas） | 部署于工厂内网 Linux 服务器，提供管理面板、API 及 WebSocket 实时通信 |
| 访客端（visitorapp） | Android 应用，安装于访客手机，通过无障碍自动化完成入场管控与离场恢复 |
| 管理面板（Dashboard） | 浏览器端管理界面，门卫用于办理入场、监控状态、办理离场 |

### 1.2 试点边界

| 项目 | 约定 |
|------|------|
| 试点区域数量 | 1 个 |
| 门卫人数 | 1–2 名 |
| 覆盖手机品牌 | 小米（Xiaomi）、OPPO、vivo、三星（Samsung） |
| 网络环境 | 工厂内网局域网（WiFi 同网段） |
| 部署模式 | 单服务器单实例 |

### 1.3 不在本次试点范围内

- 公网 / HTTPS 访问
- 多厂区 / 多区域并行管控
- 付费订阅功能
- iOS 设备支持
- 访客人脸识别或身份证核验

| 等级 | 定义 | 示例 |
|------|------|------|
| P0 | 核心流程阻断，无法通过任何手段恢复 | 服务器崩溃无法重启、所有手机无法入场 |

| P2 | 体验问题，不影响核心流程 | 界面文案不准确、操作响应略慢 |
