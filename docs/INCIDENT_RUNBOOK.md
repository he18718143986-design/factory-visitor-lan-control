# 现场故障排查手册

> **适用系统**：工厂访客手机摄像头管控系统 PoC Freeze v1  
> **面向读者**：驻场实施工程师 · 门卫主管  
> **使用场景**：现场访客入场 / 离场过程中出现异常时，按本手册排查  

---

## 一、文档用途

本手册收录系统上线后最常见的 9 类现场故障，每项均按「现象 → 根因 → 排查步骤 → 临时恢复 → 是否需要研发」五段式编写。

**使用原则**：

1. 先看「现象」匹配故障，再看「排查步骤」逐条执行
2. 每步操作后观察结果，不要跳步
3. 临时恢复方法能让访客继续流程，但不一定修复根因
4. 标注「需联系研发」的问题请在恢复后记录并上报

---

## 二、紧急联系流程

```
门卫发现异常
    │
    ├── 能按本手册恢复 ──→ 记录到「问题登记表」，当天汇总
    │
    └── 无法恢复（≥ 5 分钟）
          │
          ├── 联系驻场工程师：______________ 电话：______________
          │
          └── 工程师无法解决（≥ 30 分钟）
                │
                └── 联系研发支持：______________ 电话：______________
                    提供：故障现象 + 手机品牌型号 + 服务器 IP + 发生时间
```

**原则**：访客入场受阻超过 5 分钟时，门卫应先用备选方案（手动登记）放行，事后补录。

## 三、故障排查清单

### 故障 1：App 一直显示"正在验证"

**现象**  
访客手机扫码入场后，App 停留在转圈 +「正在验证，请稍候」画面超过 3 分钟，不进入管控状态。

**根因**  
无障碍自动化流程卡在某一步（打开开发者选项 / 开启无线调试 / 二维码配对），常见原因：
- 手机系统弹窗（密码验证、权限确认）未被自动处理
- Settings App 布局与已知 Profile 不匹配
- 全局自动化超时（180 秒后静默终止）

**排查步骤**

1. 在 App 等待页面 **长按提示文字 5 秒** 进入技术模式（Toast 提示「已进入技术模式」）
2. 观察技术模式下的状态信息：
   - 如果显示「连接未就绪，请重试或查看说明」→ 跳到 **故障 6（WebSocket 无法连接）**
   - 如果显示自动化步骤进度 → 观察卡在哪一步
3. 切到手机「设置」App，手动检查：
   - 「开发者选项」是否已开启？
   - 「无线调试」是否已打开？
   - 是否停在密码 / PIN 验证页面？如果是，让访客输入锁屏密码
4. 如果「无线调试」已开启，回到 App 技术模式，点击「重试连接」按钮

**临时恢复**

- 技术模式下点击「打开系统设置」→ 手动开启开发者选项和无线调试 → 回到 App 等待自动配对
- 如果手动操作后仍无法配对：在 Dashboard 上为该访客点击「重新生成配对码」

**是否需要联系研发**  
如果同一品牌手机连续 2 台出现此问题，记录手机型号和系统版本，联系研发。

### 故障 2：App 提示"设置已中断 / 如需帮助请联系门卫"

**现象**  
App 等待页面从「正在验证」变为「设置已中断，如需帮助请联系门卫」。

**根因**  
系统后台杀死了无障碍服务（`SetupAutomationService`），常见于内存紧张或手机省电策略激进。系统每 10 秒轮询一次，检测到服务不存在后显示此提示。

**排查步骤**

1. 打开手机「设置」→「无障碍」（或「辅助功能」）
2. 找到「Factory Control」服务，检查是否为「已关闭」
3. 如果已关闭 → 重新开启
4. 回到 App，观察是否自动恢复自动化流程

**临时恢复**

1. 重新开启无障碍服务后，App 会在下一次轮询（10 秒内）检测到并重新启动自动化
2. 如果重新开启后仍无效，强制关闭 App → 重新打开 → 重新扫码入场

**预防措施（告知访客）**

- 不要在入场过程中切出 App 到其他应用
- 不要手动在设置中关闭无障碍服务

**是否需要联系研发**  
如果同一台手机反复出现（≥ 3 次/天），记录型号和系统版本，可能需要为该机型增加保活策略。

### 故障 3：三星手机无线调试打不开

**现象**  
三星（Samsung）手机自动化流程在「开启无线调试」步骤卡住或循环重试。

**根因**  
Samsung One UI 的设置页面布局与标准 AOSP 有差异：
- 「版本号」位于「软件信息」子页面中（多一层嵌套）
- 某些 One UI 版本的开关控件 resource-id 不同
- 系统已内置 SamsungProfile 适配，但不覆盖所有 One UI 版本

**排查步骤**

1. 确认手机型号和 One UI 版本（设置 → 关于手机 → 软件信息）
2. 长按 App 等待提示 5 秒进入技术模式
3. 手动进入「设置」→「开发者选项」检查是否已打开
4. 如果开发者选项未开启：
   - 设置 → 关于手机 → **软件信息** → 连续点击「编译编号」7 次
5. 如果开发者选项已开启但无线调试未打开：
   - 设置 → 开发者选项 → 找到「无线调试」→ 手动打开
6. 回到 App 等待自动配对

**临时恢复**

手动完成以上步骤后，App 应自动检测到无线调试已开启并发起二维码配对。

**是否需要联系研发**  
记录具体 One UI 版本号（如 One UI 6.1），反馈给研发以更新 SamsungProfile。

### 故障 4：APK 下载失败

**现象**  
访客手机浏览器打开引导页后，点击下载 APK 无反应、报错或下载到 0 字节文件。

**根因**  
- `public/factory-control.apk` 文件不存在或文件名不正确
- 部分手机浏览器拦截 APK 下载
- 手机与服务器不在同一网段

**排查步骤**

1. 在服务器上确认文件存在：
   ```bash
   ls -lh /opt/nocameras/factory-saas/public/factory-control.apk
   ```
2. 确认文件大小合理（应 ≥ 10 MB）
3. 在服务器上测试 HTTP 访问：
   ```bash
   curl -sI http://localhost:3000/factory-control.apk | head -5
   ```
   期望 HTTP 200，Content-Length 与文件大小一致
4. 确认手机浏览器地址栏输入的是 `http://<服务器IP>:3000/welcome-bridge`（注意 http 而非 https）

**临时恢复**

- 换用 Chrome 浏览器重试下载
- 如果浏览器持续拦截：在手机浏览器地址栏直接输入 `http://<服务器IP>:3000/factory-control.apk`
- 最后手段：通过 USB 数据线或文件传输工具将 APK 传到手机安装

**是否需要联系研发**  
不需要，这是部署 / 网络问题。

### 故障 5：Dashboard 打不开

**现象**  
浏览器输入 `http://<IP>:3000/dashboard` 后白屏、无法加载或显示登录页但登录失败。

**根因**  
- 服务器进程未运行
- 端口未放行
- 浏览器缓存了旧的 CSRF token
- 密码错误

**排查步骤**

1. 在服务器上检查进程：
   ```bash
   pm2 status
   ```
   如果 factory-saas 不是 `online` → 执行 `pm2 restart factory-saas`
2. 检查端口：
   ```bash
   ss -tlnp | grep 3000
   ```
3. 查看最近日志是否有报错：
   ```bash
   pm2 logs factory-saas --err --lines 30
   ```
4. 如果能打开登录页但登录失败：
   - 确认邮箱 / 密码正确
   - 清除浏览器 Cookie 后重试（CSRF token 过期可能导致 403）
   - 尝试无痕 / 隐私模式打开

**临时恢复**

```bash
# 重启服务
pm2 restart factory-saas

# 如果持续失败，查看详细错误
pm2 logs factory-saas --lines 50
```

**是否需要联系研发**  
如果重启后仍无法访问，检查日志中是否有 `❌` 错误信息并截图发给研发。

### 故障 6：WebSocket 无法连接

**现象**  
Dashboard 右下角连接指示灯为红色 / 灰色，会话状态不实时更新（需手动刷新才能看到变化）。

**根因**  
- 服务器端 WebSocket 进程异常
- 网络中间设备（路由器 / 防火墙）中断了长连接
- Dashboard 页面长时间挂起后浏览器休眠了 WebSocket

**排查步骤**

1. 刷新 Dashboard 页面（F5），等待 5 秒观察指示灯是否转绿
2. 如果仍为红色，在服务器上检查：
   ```bash
   pm2 status
   ss -tlnp | grep 3000
   ```
3. 在服务器上测试 WebSocket：
   ```bash
   # 如已安装 wscat
   wscat -c ws://localhost:3000
   ```
4. 检查工厂路由器是否有 TCP 长连接超时设置（常见默认 5 分钟）

**临时恢复**

- Dashboard 内置自动重连（断开后 3–4 秒自动重试），通常刷新页面即可
- 如果服务器进程异常：
  ```bash
  pm2 restart factory-saas
  ```
- 刷新浏览器，确认转绿

**是否需要联系研发**  
如果日志中出现 `ws connection error` 或大量 `close` 事件，联系研发。

### 故障 7：访客重复扫码被拒绝

**现象**  
访客手机 App 提示「当前正在使用中，请先完成本次流程」，无法扫新的入场码。

**根因**  
这是**正常安全行为**。系统防止在 WAITING / RESTRICTED / EXITING 状态下重复扫码覆盖当前会话。

**排查步骤**

1. 确认这是不是同一个访客的同一次入场 → 如果是，无需操作，App 正在正常工作
2. 如果访客需要重新入场（例如上一次入场出了问题）：
   - 先在 Dashboard 上找到该访客的会话
   - 点击「强制离厂」让会话进入 `exited` 状态
   - 等 App 显示「离场已办理」并自动回到空闲界面（约 5–15 秒）
   - 然后重新扫码入场

**临时恢复**

如果 App 界面卡住无法操作：
1. 强制关闭 App（从最近任务列表划掉）
2. 重新打开 App
3. 如果 App 自动恢复了旧会话（12 小时内的会话会自动恢复），先在 Dashboard 上点「强制离厂」
4. 等 App 回到空闲界面后重新扫码

**是否需要联系研发**  
不需要，这是系统设计行为。

### 故障 8：离场后摄像头未恢复

**现象**  
访客扫离场码完成离场，App 显示「离场已办理，感谢配合」，但打开手机相机 App 仍然无法拍照或黑屏。

**根因**  
- ADB 连接在离场流程中已断开，管控解除命令未送达
- 手机在管控期间断网导致 ADB 连接丢失
- 设备管理器权限未正常移除

**排查步骤**

1. 确认 Dashboard 上该会话状态是否为 `exited`
2. 如果为 `exited` 但相机仍不可用：
   - 让访客打开手机「设置」→「开发者选项」→「无线调试」→ 关闭再重新打开
   - 手机重启（通常可恢复）
3. 如果 Dashboard 上状态仍为 `restricted` 或 `exiting`：
   - 点击该会话的「强制离厂」按钮
   - 等待系统执行管控解除
4. 检查日志中是否有设备断连记录：
   ```bash
   pm2 logs factory-saas --lines 50 | grep "设备不可达\|disconnect\|adb"
   ```

**临时恢复**

**最快恢复方法**：让访客重启手机。重启后系统管控策略自动失效，相机恢复正常。

如果不方便重启：
1. 设置 → 开发者选项 → 关闭「开发者选项」总开关
2. 设置 → 应用 → Factory Control → 强制停止
3. 设置 → 无障碍 → Factory Control → 关闭

**是否需要联系研发**  
如果「强制离厂」后日志显示「设备不可达，管控未解除」，且重启手机仍无法恢复相机，联系研发（极罕见）。

### 故障 9：WiFi 切换导致会话失联

**现象**  
访客手机在管控状态中切换了 WiFi（或 WiFi 断开重连），Dashboard 上显示该会话状态不再更新，App 界面仍在管控中但服务器失去连接。

**根因**  
- 手机 IP 地址因 WiFi 重连而改变，ADB 连接基于原 IP 断开
- WebSocket 连接中断后 App 端和服务端均在各自重试
- App 会话数据保存在 SharedPreferences 中，12 小时内可恢复

**排查步骤**

1. 确认手机已重新连接到正确的工厂 WiFi（与服务器同一网段）
2. 在 Dashboard 上检查该会话状态：
   - 如果为 `restricted`：管控仍生效，只是通信断了
   - 如果为 `paired_not_connected`：服务器已检测到断连
3. 如果会话状态为 `paired_not_connected`，在 Dashboard 上找到该会话点击「重试连接」
4. 在 App 技术模式（长按 5 秒）下点击「重试连接」

**临时恢复**

- 大多数情况下，WiFi 恢复后 App 会在数秒内自动重连
- 如果 1 分钟后仍未恢复，在 App 技术模式下点击「重试连接」
- 如果彻底无法恢复，在 Dashboard 上「强制离厂」→ 重新走入场流程

**是否需要联系研发**  
不需要。但如果工厂 WiFi 频繁断连，建议客户优化 WiFi 覆盖或为管控系统配置独立网络。

## 四、快速判断表

| 访客看到的画面 | 对应故障 | 门卫第一步操作 |
|---------------|---------|---------------|
| 转圈 +「正在验证，请稍候」超过 3 分钟 | 故障 1 | 长按进入技术模式查看详情 |
| 「设置已中断，如需帮助请联系门卫」 | 故障 2 | 去手机设置中重新开启无障碍服务 |
| 「当前正在使用中，请先完成本次流程」 | 故障 7 | 确认是否需要 Dashboard 强制离厂 |
| 「准备中，请稍候」持续不动 | 故障 1 / 6 | 长按技术模式 → 检查 WiFi 连接 |
| 离场后相机仍不可用 | 故障 8 | 让访客重启手机 |
| App 下载不了 | 故障 4 | 换 Chrome 浏览器或直接输入 APK 地址 |

## 五、服务器端快速诊断命令

在遇到任何故障时，可在服务器上依次执行以下命令收集信息：

```bash
# 1. 检查服务是否存活
pm2 status

# 2. 检查端口
ss -tlnp | grep 3000

# 3. 查看最近错误日志
pm2 logs factory-saas --err --lines 30

# 4. 查看最近正常日志
pm2 logs factory-saas --out --lines 30

# 5. 重启服务（万能恢复）
pm2 restart factory-saas

# 6. 检查数据库文件
ls -lh data/factory.db

# 7. 检查磁盘空间
df -h /opt/nocameras

# 8. 检查内存
free -h
```
